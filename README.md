## 📌 1. 프로젝트 개요

낯선 장소(여행지, 출장지, 파견 등)에서 식사할 곳을 고를 때 **"뭘 먹을지 고민되는 상황"**이나 **"결정 장애"**를 겪는 사용자를 위한 모바일 애플리케이션입니다. 

이번 v4 업데이트에서는 기존의 단일 카드 UI 형태에서 **네이버 지도 및 카카오 맵 스타일의 세련된 풀스크린 지도 및 Persistent Bottom Sheet 기반 모던 UX**로 대폭 개편했습니다. 

사용자의 **현재 실시간 GPS 위치**를 기반으로 반경 2km 이내의 음식점을 검색하고, 사용자의 **인원 수, 예산, 기분, 식사 상황** 정보를 입력받아 **Google Gemini AI**가 최적의 맛집 최대 3개를 엄선하여 그 구체적인 추천 이유와 함께 제안합니다.

### 🎯 주요 타겟 사용자
* 처음 가본 여행지나 출장지에서 빠르게 식사해야 하는 사용자
* 회사 미팅, 세미나, 동창회 등 특정 예산과 상황에 맞는 식당을 찾아야 하는 모임 주최자
* 주변 식당 정보는 많지만, 오늘 내 기분과 상황에 딱 맞는 숨은 맛집을 골라보고 싶은 사용자

## ✨ 3. 핵심 기능 및 특징

### 1️⃣ 실시간 위치 기반 풀스크린 카카오 맵 연동 (FR-01, FR-02)
* `FusedLocationProviderClient`를 사용해 사용자의 **실시간 GPS 경위도 좌표**를 가져옵니다.
* **WebView 기반 카카오 맵 JS API**를 연동하여 앱을 켬과 동시에 내 주변 지도를 로드합니다.
* 내 위치 및 AI가 추천한 맛집 3선의 좌표를 지도상에 마커로 매핑하고 마커 클릭 시 안드로이드 네이티브 식당 상세 정보 팝업창을 띄워주는 브릿지(`JavascriptInterface`)가 구축되어 있습니다.
* 에뮬레이터 환경에서 그래픽 가속 에러로 지도가 노출되지 않는 버그를 우회하기 위해 소프트웨어 가속(`LAYER_TYPE_SOFTWARE`) 및 300ms 지연 레이아웃 갱신(`map.relayout()`) 보정 알고리즘을 적용했습니다.

### 2️⃣ Persistent Bottom Sheet 기반 단계별 추천 마법사 UX (FR-03)
* 구글 공식 머티리얼 컴포넌트인 `BottomSheetBehavior`를 활용하여 아래에서 끌어올리는 슬라이드 바를 구현했습니다.
* [오늘의 AI 맛집 추천 받기 🚀] 플로팅 버튼 터치 시 Bottom Sheet가 활성화되어 인원 → 예산 → 기분 → 상황 단계의 마법사형 조건 입력을 받습니다.
* 마법사 진행 도중 사용자가 화면을 슬라이드해서 의도치 않게 입력창이 닫히는 현상을 예방하기 위해 `peekHeight = 0` 및 완전 확장 모드(`STATE_EXPANDED`)를 고수하도록 제어했습니다.

### 3️⃣ 결과 분석 완료 후 맵-리스트 동시 노출 (STATE_COLLAPSED)
* AI 분석이 완료되면 Bottom Sheet 높이가 자동으로 `240dp`인 접힘 상태(`STATE_COLLAPSED`)로 내려가 하단에는 추천 맛집 리스트를, 상단에는 지도 마커들의 위치를 동시에 보여주어 최상의 탐색 편의성을 자랑합니다.
* 사용자가 하단 바를 위로 스르륵 올리면 상세 리스트를 스크롤하며 정보를 볼 수 있고, 바를 완전히 아래로 숨기면 지도만 넓게 볼 수 있도록 복구됩니다.

### 4️⃣ Google Gemini AI 연동 개인화 추천 (FR-04, FR-05)
* 카카오 Local API 검색 결과와 사용자 선택 조건을 프롬프트로 가공하여 최신 **gemini-2.5-flash** 모델에 실시간으로 전달합니다.
* LLM의 환각(Hallucination) 현상을 극복하기 위해 **강력한 프롬프트 제약 조건** 및 **폴백 안전장치(검색된 목록 상위 3개 대체 매칭)**를 설계하여 존재하지 않는 식당이 추천되는 문제를 방지했습니다.

### 5️⃣ API 키 동적 설정 및 보안 (🔑 API 설정)
* 소스코드 내 API 키 하드코딩을 원천 차단하여 깃허브 업로드 시의 키 노출 사고를 방지했습니다.
* 우상단 톱니바퀴 버튼을 통해 **카카오 REST API 키**, **카카오 JavaScript 키**, **Gemini API 키**를 동적 기입하며, 암호화 및 SharedPreferences를 통해 로컬 기기에만 저장합니다.
* `AndroidManifest.xml` 내에 `usesCleartextTraffic="true"` 설정을 주입해 카카오 이미지 타일 리소스(HTTP)의 통신 차단을 해제했습니다.

## 🚀 5. 실행 및 테스트 방법 (중요)

이 프로젝트는 **보안 가이드라인**에 따라 카카오 및 Gemini API 키가 소스코드에 포함되어 있지 않습니다. 앱 실행 및 테스트를 위해 아래 단계를 진행해 주세요.

### Step 1. API 키 발급받기
1. [카카오 개발자 센터](https://developers.kakao.com/)에서 애플리케이션을 등록하고 **REST API 키**와 **JavaScript 키**를 발급받습니다.
   - **중요**: [내 애플리케이션] > [플랫폼] > [Web] 설정에 사이트 도메인으로 `http://localhost` 및 `https://localhost`를 등록해야 웹뷰 지도가 정상적으로 작동합니다.
2. [Google AI Studio](https://aistudio.google.com/)에서 **Gemini API 키**를 발급받습니다.

### Step 2. 앱 실행 및 키 등록
1. 프로젝트를 안드로이드 스튜디오에서 열고 에뮬레이터 또는 실기기에 빌드하여 실행합니다.
2. 앱 메인 화면 우측 상단의 **⚙️ (API 설정)** 아이콘을 클릭합니다.
3. 발급받은 **Kakao REST API 키**, **Kakao JavaScript 키**, **Gemini API 키**를 입력하고 **[저장]** 버튼을 누릅니다.

### Step 3. 맛집 추천받기
1. 위치 권한이 승인되면 지도가 자동으로 현재 사용자 한글 주소 중심으로 갱신됩니다.
   - *에뮬레이터 사용 시*: 위치가 미국으로 감지되면 경고창 가이드에 따라 에뮬레이터 위치 탭에서 서울 좌표(위도 `37.5665`, 경도 `126.9780`)를 설정한 뒤 [Set Location]을 클릭하고 다시 조회 버튼을 누릅니다.
2. 하단의 **[오늘의 AI 맛집 추천 받기 🚀]** 버튼을 누르면 Bottom Sheet가 열리며 4단계 조건 마법사가 차례로 나타납니다.
3. 인원, 예산, 기분, 상황 조건을 각각 선택(또는 ✏️ 직접 입력)합니다.
4. 최종 요약 확인 후 **[AI 맛집 추천 시작하기]** 버튼을 누르면 추천 카드가 바 내부에 생성되고, 바가 접힌 상태(`STATE_COLLAPSED`)로 미끄러져 내려오며 지도 상에 음식점 위치 핀 마커가 노출됩니다.
5. 마커 또는 리스트 아이템을 누르면 상세 주소, 전화번호, 거리, 그리고 Gemini가 분석한 **자세한 추천 이유**를 확인하고 카카오맵 상세 정보 페이지로 바로 이동할 수 있습니다.
